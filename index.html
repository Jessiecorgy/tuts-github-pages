Introduction

I’m sure that a lot of you who have created WordPress themes or plugins have created documentation for them. A lot of times that also means setting up another domain and hosting. There’s also creating the site, maybe using your theme, and managing the content. It’s all of these steps that can add to reasons why the documentation step happens last or not at all.

What if it could be easier? What is you could keep your documentation in the same repository as your project’s code? It definitely can be if you are using GitHub to host your code. Let’s talk about GitHub Pages.

This is the first post in the series to talk about GitHub Pages. I will give my thoughts about why it’s important to keep your documentation with your code. I will also give a little overview on what GitHub Pages is and how to enable it for your project.

What is GitHub Pages?

GitHub Pages is a public website hosting service provided by GitHub. You get can have one website for your organization or user and unlimited sites for your projects. You can use GitHub Pages for both public and private repositories.

The pages are served as HTTP, not HTTPS. That is just something to keep in mind to not use this for anything you want encrypted like passwords or credit card numbers.

There are a number of popular projects that are hosted on GitHub Pages, and you more than likely have viewed them. If you have looked at the documentation for Bootstrap, Font Awesome, or Yeoman, then you have visited a site that is hosted on GitHub Pages.

The sites load crazy fast and your changes are visible quickly. It really is true what they say,

Hosted directly from your GitHub repository. Just edit, push, and your changes are live.

Enabling GitHub Pages

There are a number of ways to set up GitHub Pages for your project. You can generate a site via GitHub Pages or you can start from scratch. For this post we are going to generate a site and in the next post we will start from scratch. You can follow along here, just make sure to select “Project site” and “Generate a site”. Also make sure to create a new GitHub repository or use an existing one. You can view my example project as well.

The first thing we need to do is enable GitHub Pages for your repository. You can do this by going to “Settings”, located in the menu on the right.

The next thing we will want to do is scroll down to the “GitHub Pages” section and click on the “Automatic page generator” button.

Now you will be able to edit your first page. Make sure to add an appropriate Title and Tagline. You can also use GitHub Flavored Markdown to edit the contents of the page in the Body section. Once you have your content ready, scroll down to the bottom of the page.

Be sure to notice the “Google Analytics Tracking ID” option. If this is something that you want to track page views, be sure to create a new tracking ID and paste it in that field (ex: UA-XXXXXXXX-X).

Once you have finished getting everything where you want it for your first page, go ahead and click the “Continue to layouts” button.

Now we will be able to select a layout for our site. The default layout is “Slate”, but there are many others you can choose from.

The “Minimal” layout is my favorite, so I chose to use it.

Once you choose the layout you want, simply click the “Publish” button. You will then be directed back to your project and there will be a little alert that will give you a link to your new site. Go ahead and click on that link, but do know that it can take up to 10 minutes for your site to be generated.
